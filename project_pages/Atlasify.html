Atlasify

Machine Learning and User Interface Improvements

I worked on the Atlasify research project with Prof. Doug Downey. The platform is designed to give explanations about how subjects are related to locations. The platform pulls descriptions from Wikipedia and uses Machine Learning to pick the best descriptions.

Contribution

I designed the interface to increase user feedback and added multiple classifiers to the Machine Learning Algorithim

Interaction

In the initial popup (bordered in red), users had to choose a relation and then click submit. There was also a why box which was not necessary to fill out that detered users from submitting any feedback during testing. The machine learning algorithim could not interpret the reason a user typed in, so it was really only necessary to get the number relation data from users to improve the algorithim.

Algorithim

My idea for improving the algorithim was to create history and media classifiers. I noticed a lot of the descriptions that were historical were "good" definitions. I also noticed that the descriptions from fictional books and other fictional media sources were poor descriptors because they were describing relations that weren't true.

I created and trained bag of words history and media classifiers using scikit-learn. The media classifier was difficult to properly train because of the low number of media descriptions, but the history one was more accurate. I also incorporated Semantic Relatedness into the algorithm. This is the measure of Wikipedia pages you would have to click through to get to one page from another. A description whose subject has a low SR to both search topics would indicate a closer real world relation to those search topics.

Using user inputed realatedness values, I trained a classifier that took in history, media, and SR relatedness and could predict a new relatedness value.
